# Структура проекта голосового ассистента

## Корневые файлы
- `main.py` — консольная версия ассистента. Запускает диалог в терминале. Используется для быстрой отладки логики без графики.
- `run_assistant.py` — точка входа для графической версии. Запускает приложение с иконкой в трее (через `gui/tray.py`).
- `requirements.txt` — список всех необходимых Python-библиотек с точными версиями.
- `.gitignore` — список файлов и папок, которые не должны попадать в git (виртуальное окружение, кэш, модели и т.д.).
- `README.md` — основная информация о проекте (назначение, установка, запуск).
- `STRUCTURE.md` — этот файл, объясняющий структуру проекта.

## Папка `assistant/` — логика ассистента (не зависит от интерфейса)
- `__init__.py` — делает папку Python-пакетом (может быть пустым).
- `config.py` — настройки: URL Ollama, имя модели, пути к моделям Vosk/Silero и т.д.
- `utils.py` — вспомогательные функции (логирование, автозагрузка и пр.).
- `llm_client.py` — класс `LlamaClient` для общения с Ollama через HTTP. Содержит методы `send_message`, `send_messages`, `check_connection`.
- `core.py` — класс `Assistant`, который хранит историю диалога и использует `LlamaClient` для получения ответов. Методы: `process_message`, `clear_history`.
- `tools.py` — (будет позже) реестр инструментов, которые может вызывать модель (например, открыть проводник).
- `system_commands.py` — (будет позже) функции для выполнения системных команд (открыть программу, создать папку и т.п.).
- `stt.py` — (будет позже) распознавание речи через Vosk.
- `tts.py` — (будет позже) синтез речи через Silero.
- `speaker_verification.py` — (будет позже) идентификация говорящего (SpeechBrain).

## Папка `gui/` — графический интерфейс (PySide6)
- `__init__.py` — делает папку пакетом.
- `main_window.py` — главное окно чата. Содержит класс `ChatWindow` (наследник QMainWindow) с историей сообщений, полем ввода, кнопкой. Также здесь определён класс `AssistantWorker` (QThread) для асинхронного вызова модели без подвисания интерфейса.
- `tray.py` — иконка в системном трее. Класс `TrayApp` (наследник QSystemTrayIcon) создаёт меню, позволяет показать/скрыть окно, выйти из приложения. Пока содержит заглушку для голосового ввода.
- (опционально) `resources/` — папка для иконок, стилей и т.п.

## Как взаимодействуют файлы

1. Пользователь запускает `run_assistant.py`.
2. Создаётся экземпляр `QApplication`, затем `TrayApp` (иконка в трее).
3. `TrayApp` создаёт `ChatWindow` (но не показывает его сразу).
4. В `ChatWindow` создаётся экземпляр `Assistant` из `assistant.core`.
5. При отправке сообщения `ChatWindow` создаёт `AssistantWorker`, который в отдельном потоке вызывает `assistant.process_message()`.
6. `Assistant.process_message()` добавляет сообщение в историю, отправляет запрос через `LlamaClient`, получает ответ, сохраняет его и возвращает.
7. `AssistantWorker` возвращает ответ через сигнал, и он отображается в окне.

## Запуск
- Для консольной версии: `python main.py`
- Для графической версии: `python run_assistant.py`

## Важные замечания
- Убедись, что в папках `assistant` и `gui` есть файлы `__init__.py` (пустые) — иначе Python не сможет импортировать модули.
- Для корректной работы иконки в трее нужен файл иконки (например, `icon.png`). Если его нет, временно закомментируй строку `self.setIcon(...)` в `tray.py`.
